## 问题1：人脸注册报错 "No static resource face/check"

### 现象
- 点击"人脸注册"按钮，弹出错误：`系统内部错误: No static resource face/check.`
- 但人脸注册实际上成功了，百度智能云中能看到注册的人脸

### 原因分析
- `ShiroConfig.java` 中 `/face/check` 接口没有配置为匿名访问
- 现有配置：
  ```java
  filterChainDefinitionMap.put("/face/config", "anon");     // 已配置
  filterChainDefinitionMap.put("/face/register", "jwt");    // 已配置
  filterChainDefinitionMap.put("/face/verify", "jwt");      // 已配置
  ```
- 缺少：`/face/check` 的配置

### 解决方案
在 `ShiroConfig.java` 中添加：
```java
filterChainDefinitionMap.put("/face/check", "anon");
```

### 方案对比

| 方案 | 优点 | 缺点 | 难度 |
|------|------|------|------|

- 根据商品名称和分类生成商品图片

**方案B：电商图片API（备选）**
- 使用合规的图片API服务（如 Unsplash、Pexels）
- 搜索商品相关图片
- 免费且无版权风险

## 问题4：会员余额和积分真实扣减

### 现状
- 当前代码只有**积分累加**，没有**余额扣减**
- `MemberPointsConsumer.java` 中积分计算逻辑：`积分 = 实付金额 / 10`

### 需要新增的逻辑

#### 4.1 会员余额扣减
```java
// PaymentServiceImpl.java 或 OrderServiceImpl.java
public void deductMemberBalance(String memberId, BigDecimal amount) {
    Member member = memberService.findByMemberId(memberId);
    if (member == null) {
        throw new RuntimeException("会员不存在");
    }
    if (member.getBalance().compareTo(amount) < 0) {
        throw new RuntimeException("余额不足");
    }
    member.setBalance(member.getBalance().subtract(amount));
    memberService.updateMember(member);
}
```

#### 4.2 支付方式选择
- 现金/微信/支付宝/银行卡：不扣减会员余额，只累加积分
- **会员余额支付**：扣减会员余额，累加积分
- 刷脸支付：识别会员后，可选择扣减余额

#### 4.3 余额支付流程
```
1. 用户选择"余额支付" → 检查会员余额是否充足
2. 余额充足 → 扣减余额 → 创建订单 → 累加积分
3. 余额不足 → 提示用户更换支付方式或充值
```

| `checkout/index.vue` | 新增"余额支付"选项，显示会员余额 |

| `OrderService.java` | 支付成功后更新会员余额 |
